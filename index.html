一句话总览

Claude Code 本质上是一个“终端 UI + 对话状态机 + 工具编排器”

入口从 entrypoints/cli.tsx 开始，真正的大脑在 main.tsx 做初始化与装配；交互 UI 由 Ink/React 驱动； 一次请求先被 processUserInput 预处理，再进入 query() 的流式循环；当模型返回 tool_use 时，由工具编排器执行 Bash、Read、Edit、MCP 等工具，再把结果回灌给模型继续推理。

你真正要记住的分层

  1. 启动层：CLI 参数解析、模式分流、配置和环境准备。
  2. 装配层：工具池、commands、skills、plugins、MCP 客户端、agent 定义。
  3. 执行层：REPL 或 headless QueryEngine 发起一轮对话。
  4. 编排层：query 循环处理模型流式输出、工具调用、压缩和恢复。
  5. 扩展层：MCP、Skill、Plugin、Coordinator、Remote/Bridge 叠加能力。

    Reading Model

    用一句话理解每条链路

    启动

    entrypoints/cli.tsx 先做快路径分发，真正复杂的初始化在 main.tsx 中完成。

    交互

    launchRepl() 把 Ink 的根组件、AppState 和 REPL 屏幕拼起来，形成终端应用。

    请求

    用户输入不会直接打给模型，而是先过 slash command、hook、附件、权限和上下文注入。

    工具

    模型输出中只要出现 tool_use，Claude Code 就会进入工具执行回路，而不是立刻结束。

    扩展

    Skill、Plugin、MCP 都不是平行体系，它们最终都会变成模型可见的命令、工具、资源或上下文。

    多 Agent

    Coordinator 模式不是换一个 UI，而是在同一套 query / tool 基础设施上，额外叠加 worker 编排规则。
